1. 定义实体完整性

1.1. 实体完整性检查和违约处理

用 PRIMARY KEY 短语定义了关系主码之后，用户每插入一条记录或者对主码列更新的时候，关系数据库将进行实体完整性规则自动进行检查：

  1. 检查主码是否唯一，如果不唯一啧拒绝插入或者修改
  2. 检查主码的各个属性是否为 NULL ，只要有一个为 NULL 就拒绝插入或者修改

检查方法：

  1. 全表扫描，依次判断每一条记录 但是十分耗时
  2. 在主码上自动建立一个索引
